# Полное описание проекта

## 1. Назначение

Проект реализует Telegram-бота для реферального розыгрыша в канале.  
Ключевая цель: увеличить подписчиков и сохранить лояльность через прозрачную механику участия.

## 2. Основные правила участия

1. Участвуют и старые, и новые подписчики канала.
2. Участвуют только пользователи, которые зарегистрировались через бота (`/start`).
3. Для участия обязателен опрос: возраст и любимый вид спорта.
4. После регистрации и опроса пользователь подтверждает подписку кнопкой `Проверить подписку`.
5. За подтвержденное участие пользователь получает 1 базовый билет.
6. Если пользователь пришел по реферальной ссылке и подтвердил подписку, пригласивший получает +1 билет.

## 3. Как устроено технически

### 3.1 Вход и регистрация

- Пользователь заходит в бота по ссылке `https://t.me/<bot_username>?start=<ref_code>` или по обычной ссылке на бота.
- Хендлер `/start`:
  - создает пользователя, если его нет;
  - обновляет username для существующего пользователя;
  - сохраняет привязку к рефереру (если payload валиден и это не self-referral);
  - выдает персональную реферальную ссылку;
  - запускает обязательный опрос, если анкета еще не заполнена.

### 3.2 Обязательный опрос

- Опрос реализован через FSM:
  - шаг 1: ввод возраста (`SurveyState.waiting_age`);
  - шаг 2: ввод вида спорта (`SurveyState.waiting_sport`).
- Ответы сохраняются в таблице `users`:
  - `age`;
  - `favorite_sport`.

### 3.3 Проверка подписки

- Кнопка `Проверить подписку` вызывает `getChatMember`.
- Пользователь считается подписанным при статусах: `member`, `administrator`, `creator`.
- Если анкета не заполнена, участие не подтверждается.

### 3.4 Начисление билетов

Сервис `ReferralService.verify_and_award` делает 2 независимые операции:

1. **Базовый билет за участие**  
   - Начисляется самому пользователю после успешной проверки подписки и анкеты.
   - Причина: `registration_verified`.

2. **Реферальный билет**  
   - Начисляется пригласившему, если у текущего пользователя есть реферальная заявка в активной кампании.
   - Причина: `referral_verified`.

Для идемпотентности используется `tickets.idempotency_key`:
- `registration:<campaign_id>:<user_id>`
- `referral:<campaign_id>:<referral_id>`

Это защищает от повторных начислений при повторных нажатиях.

### 3.5 Защита от злоупотреблений

- Self-referral блокируется.
- Для одного invitee в рамках кампании допускается одна реферальная запись.
- Есть rate limit на `/start` и `Проверить подписку`.
- Все важные действия пишутся в `audit_logs`.

### 3.6 Розыгрыш

- Для активной кампании накапливаются билеты в `tickets`.
- `DrawService` строит «пул» шансов: каждый билет увеличивает количество вхождений пользователя.
- Победитель выбирается случайно (`secrets.choice`), результаты фиксируются в `draw_results`.
- По расписанию (`APScheduler`) выполняется автоматическая финализация завершенных кампаний.

## 4. Структура проекта

- `app/main.py` — запуск бота, подключение роутеров и планировщика.
- `app/handlers/` — пользовательские и админские команды.
- `app/services/` — бизнес-логика (рефералы, подписка, кампании, розыгрыш).
- `app/repositories/` — доступ к БД.
- `app/db/models.py` — ORM-модели.
- `app/db/migrations/0001_init.sql` — базовая схема.
- `app/db/migrations/0002_participation_and_survey.sql` — расширение под опрос и идемпотентность.

## 5. Модель данных

### users
- `telegram_id`, `username`, `ref_code`, `referred_by`
- `age`, `favorite_sport`
- `is_blocked`, `created_at`

### campaigns
- период кампании, статус, параметры механики

### referrals
- связь inviter -> invitee в контексте кампании
- статусы `pending/verified/rejected`

### tickets
- все начисленные шансы с причиной и idempotency key

### draw_results
- победители и техническая информация о выборе

### audit_logs
- журнал действий для проверки спорных кейсов

## 6. Полная механика проведения розыгрыша

1. Админ создает кампанию (`/campaign_new ...`) и активирует ее (`/campaign_start <id>`).
2. В канал публикуется анонс с призом и ссылкой на бота.
3. Пользователь нажимает `/start`, получает ссылку, проходит опрос.
4. Пользователь подписывается на канал и нажимает `Проверить подписку`.
5. Бот:
   - проверяет подписку;
   - проверяет, заполнен ли опрос;
   - начисляет базовый билет участнику;
   - начисляет +1 билет пригласившему (если применимо).
6. До конца кампании пользователи приглашают друзей.
7. После завершения кампании выполняется розыгрыш (`/draw <id>` или авто-финализация).
8. Публикуются итоги, а новым подписчикам дается welcome-цепочка контента.

## 7. Что важно для честности и удержания

- Честность:
  - открытые правила;
  - фиксируемые действия в БД;
  - отсутствие повторных начислений;
  - аудит логов.
- Удержание:
  - релевантный приз по теме канала;
  - короткие и понятные шаги участия;
  - контент после розыгрыша (welcome-цепочка).
